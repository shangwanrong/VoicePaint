# AI 语音绘图工具（VoicePaint）— 项目开发文档

## 一、项目概述

### 1.1 项目目标
开发一款**纯语音控制**的绘图工具，用户无需使用鼠标或键盘，仅通过语音指令即可完成绘图创作。

### 1.2 核心约束
- **零鼠标/键盘交互**：所有绘图操作必须通过语音指令完成
- **实时响应**：语音识别到绘图执行的延迟应控制在可接受范围内
- **指令容错**：支持模糊指令、口语化表达、指令纠错

### 1.3 技术栈选型

| 层级 | 技术方案 | 说明 |
|------|----------|------|
| 前端框架 | HTML5 + CSS3 + JavaScript | 纯浏览器端应用，无需后端服务器 |
| 画布引擎 | HTML5 Canvas API | 原生绘图，性能可控 |
| 语音识别 | Web Speech API (SpeechRecognition) | 浏览器原生支持，零延迟本地识别（Chrome/Edge） |
| 语音合成 | Web Speech API (SpeechSynthesis) | 语音反馈，替代视觉提示 |
| 指令解析 | 自定义 NLU 引擎 | 基于关键词+正则+语义模板的指令理解 |
| 打包部署 | 静态 HTML 文件 | 直接打开即用，无需安装 |

### 3.2 核心模块说明

#### 模块1：语音识别管理器 (VoiceRecognizer)
- **职责**：管理 Web Speech API 的生命周期，持续监听语音输入
- **关键设计**：
  - 自动重启机制：识别结束后自动重新开始监听
  - 静音检测：长时间静音自动提交当前结果
  - 中间结果实时显示：`onresult` 中区分 `isFinal` 和中间结果
  - 错误恢复：网络断开、权限拒绝等异常处理

#### 模块2：指令解析引擎 (CommandParser)
- **职责**：将自然语言文本转换为结构化的绘图指令对象
- **解析流程**：
  1. **预处理**：去除语气词（"请"、"那个"、"嗯"等）、统一标点
  2. **意图识别**：通过关键词词典匹配指令意图
  3. **参数提取**：通过正则模板提取颜色、尺寸、位置等参数
  4. **模糊匹配**：编辑距离算法处理同义词和口误
  5. **上下文补全**：省略参数时从上下文推断默认值
- **输出格式**：
  ```javascript
  {
    intent: "draw_shape",       // 意图
    shape: "circle",            // 图形类型
    params: {                   // 参数
      x: 400, y: 300,          // 位置（默认画布中心）
      radius: 50,              // 半径
      color: "#ff0000",        // 颜色
      fill: true,              // 是否填充
      lineWidth: 2             // 线宽
    },
    raw: "画一个红色的圆",       // 原始文本
    confidence: 0.95            // 置信度
  }
  ```

#### 模块3：指令执行器 (CommandExecutor)
- **职责**：验证指令合法性，执行绘图操作，管理操作历史
- **关键设计**：
  - 操作栈：支持撤销/重做
  - 指令队列：复合指令拆解为原子操作依次执行
  - 执行反馈：每条指令执行后通过语音播报确认

#### 模块4：画布渲染器 (CanvasRenderer)
- **职责**：管理 Canvas 绘图上下文，维护图形对象列表，执行渲染
- **关键设计**：
  - 图形对象列表：每个绘制的图形保存为对象，支持删除和重绘
  - 双缓冲：离屏 Canvas 缓存已完成图形，仅重绘变化部分
  - 视图状态：维护缩放、平移等视图变换

#### 模块5：反馈系统 (FeedbackSystem)
- **职责**：通过语音和视觉向用户提供操作反馈
- **关键设计**：
  - 语音播报：使用 SpeechSynthesis 播报操作确认
  - 状态栏：显示当前画笔颜色、线宽、模式等状态
  - 光标指示：在画布上显示当前画笔位置（十字准星）

## 四、关键数据结构

### 4.1 图形对象 (ShapeObject)

```javascript
class ShapeObject {
  constructor(type, params) {
    this.id = generateId();        // 唯一标识
    this.type = type;              // 'line'|'rect'|'circle'|'triangle'|'star'|'arrow'|'text'|'freehand'
    this.params = params;          // 图形参数（位置、大小等）
    this.style = {                 // 样式
      strokeColor: '#000000',
      fillColor: '#000000',
      lineWidth: 2,
      lineDash: [],               // 虚线模式
      opacity: 1.0,
      fill: false
    };
    this.timestamp = Date.now();   // 创建时间
  }
}
```

### 4.2 指令对象 (CommandObject)

```javascript
class CommandObject {
  constructor(intent, params, raw) {
    this.intent = intent;          // 意图标识
    this.params = params;          // 提取的参数
    this.raw = raw;                // 原始语音文本
    this.confidence = 1.0;         // 置信度
  }
}
```

### 4.3 操作历史 (ActionHistory)

```javascript
class ActionHistory {
  constructor() {
    this.undoStack = [];           // 撤销栈
    this.redoStack = [];           // 重做栈
    this.maxSize = 50;             // 最大历史记录数
  }
}
```

## 六、容错与鲁棒性设计

### 6.1 语音识别容错
- **中间结果过滤**：忽略低置信度的中间识别结果，仅处理 `isFinal=true` 的最终结果
- **重复指令防抖**：500ms 内相同指令不重复执行
- **静音超时**：3秒无语音自动提交当前识别结果

### 6.2 指令理解容错
- **同义词扩展**：同一意图支持多种表达方式
- **模糊匹配**：使用编辑距离算法，允许1-2个字的偏差
- **省略补全**：缺少参数时使用合理默认值（位置默认中心，大小默认50，颜色默认当前色）
- **歧义消解**：多意图匹配时选择置信度最高的，或通过语音询问用户确认

### 6.3 执行容错
- **参数范围校验**：尺寸不能为负、坐标不能超出画布
- **操作确认**：清空画布等不可逆操作需二次语音确认
- **错误恢复**：执行失败时自动撤销，语音提示错误原因

## 七、响应延迟优化

### 7.1 延迟分析

| 阶段 | 预期延迟 | 优化策略 |
|------|----------|----------|
| 语音识别 | 200-800ms | 使用浏览器本地识别，避免网络往返 |
| 指令解析 | <10ms | 纯本地计算，关键词匹配 |
| 指令执行 | <16ms | Canvas 绘制单帧 <16ms |
| 语音反馈 | 100-300ms | 非关键路径，异步执行 |
| **端到端** | **300-1100ms** | **可接受范围** |

### 7.2 优化措施
- 语音识别使用 `continuous` 模式 + `interimResults`，实现边说边识别
- 简单指令（颜色、线宽）直接执行，无需语音确认
- 复杂指令先执行再播报确认，不等播报完成
- Canvas 渲染使用 `requestAnimationFrame`，避免阻塞

## 十一、风险与限制

### 11.1 技术限制

| 限制 | 影响 | 缓解措施 |
|------|------|----------|
| Web Speech API 浏览器兼容性 | 仅 Chrome/Edge 可靠支持 | 在页面中检测浏览器并提示用户 |
| 语音识别准确率 | 方言、口音、噪音影响识别 | 扩展同义词词典，增加模糊匹配 |
| 语音识别需要网络 | 部分浏览器需要联网才能使用语音识别 | 提示用户保持网络连接 |
| Canvas 绘图精度 | 复杂图形可能不够精确 | 使用数学计算保证精度 |
| 无后端支持 | 无法保存/加载作品到云端 | 提供本地下载功能 |
